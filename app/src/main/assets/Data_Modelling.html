    CREATING TABLES

    A table is a database object that is used to store data about a particular subject, such as students, employees or products. A table consists of records and fields.

    Each record contains data about one instance of the table subject, such as a particular employee. A record is also commonly called a row or an instance.

    Each field contains data about one aspect of the table subject, such as a first name or e-mail address. A field is also commonly called a column or an attribute.

    A record consists of field values, such as OAU, Ltd. or someone@example.com. A field value is also commonly called a fact.

    A database can contain many tables, each storing information about a different subject. Each table can contain many fields of different types of data, such as text, numbers, dates, and hyperlinks.


    TABLE AND FIELD PROPERTIES

    Tables and fields also have properties that can set to control their characteristics or behaviour.

    Table and Field Properties

    A table open in Design View

    1.  Table Properties
    2.  Field Properties

    In Access database, table properties are attributes of a table that affect the appearance or behaviour of the table as a whole. Table properties are set in the table's propety sheet, in Design view. For example, you can set a table's Default View property to specify how the table is displayed by default.

    A field property applies to a particular field in a table and defines one of the field's characteristics or an aspect of the field's behaviour. You can set some field properties in Datasheet view. You can also set any field property in Design view by using Field Properties pane.


    DATA TYPES

    Every field has a data type. A field's data type indicates the kind of data that the field stores, such as large amounts of text or attached files.

    Data Type

    A data type is a field property, but it differs from other field properties as follows:
    i. You set a field's data type in the table design grid, not in the Field Properties pane.
    ii. A field's data type determines wgat other properties the field has.
    iii. You must set a field's data type when you create the field.

    Data type denotes the type of information the field name will contain. If the field name holds a numeric value, the data type will be number.


    TABLE RELATIONSHIPS

    Although each table stores data about a different subject, tables in a database usually data about subjects that are related to each other. For exapmle, a database might contain:
    i. A customers table that lists your company's customers and their addresses.
    ii. A products table that lists the products that you sell, including prices and pictures for each item.
    iii. An orders table that tracks customer orders.

    Because storing data about differents subjects in separate tables, you need some way to tie the data together so that you easily combine related data from those separate tables. To connect the data stored in different tables, you create relationships. A relationship is a logical connection between two tables that specifies fields that the tables have in common.
